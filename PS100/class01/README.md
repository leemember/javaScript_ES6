### 📃 문제 1번 : 배열의 삭제

- 다음 배열에서 400, 500를 삭제하는 코드를 입력하세요.

```
var nums = [100,200,300,400,500];
```

배열에서 지원하는 내장함수가 있다. 바로 pop이라는 함수다.
pop이라는 함수는 배열에서 가장 끝에 있는 원소를 제거하고 그 값을 반환하는 함수이다.
console.log(nums.pop()); 이렇게 해주면 콘솔창에는

<code>100,200,300,400</code> 만 찍히는 것을 확인 해볼 수 있다.
그럼 pop이라는 것은 가장 끝에 있는 원소들을 제거해주는 함수이니 nums.pop()을 두 번 사용해주면 400,500 이 사라진다.

### 📃 문제 2번 : 배열의 내장함수

- <pass>부분에 배열 내장함수를 이용하여 코드를 입력하고 다음과 같이 출력되게 하세요.

```
데이터
var arr = [200,100,300];
//pass
console.log(arr);

출력
[200,100,10000,300]
```

✨splice✨ : 배열 안에 새로운 원소를 **삽입하거나 삭제하거나 교체하는 역할**을 한다. 이 함수는 3가지 매개변수가 필요한데 첫 번째 매개변수는 변화를 할 원소에 첫번째 인덱스를 의미한다.

<code>arr.splice(0, 0, 0);</code>
첫 번째 인덱스 값이 0이라면 arr의 200을 가르키는 것이다. 1이라면 arr 함수 배열에서 두 번째 원소 100을 의미하는거고, 2라면 세 번째 원소 300을 의미하는 것이다.

두 번째에 들어 갈 매개변수는 제거 할 원소의 갯수이다. 첫 번째가 0이고 두 번째 매개변수가 1이면 200부터 1개의 원소를 제거한다는 뜻이다. (즉, 200을 제거한다는 의미)

세번째 매게변수에 의미는 기존에 있던 원소를 삭제하고 새롭게 삽입 할 값을 의미한다.

첫번째 인덱스부터 1개를 삭제하고 10000을 집어넣으면
arr.splice(0, 1, 10000) 으로 구현하면 된다.
그럼 콘솔창에는 [10000, 100, 300] 이 찍힌다. 근데 우리의 목표는 [200, 100, 10000, 300]
이 값으로 출력되게 하는 것이니 <code>arr.splice(2, 0, 10000);</code>
으로 해주면 된다. 두 번째 매개변수가 0이면 아무것도 삭제되지 않는다.

### 📃 문제 3번 : 변수의 타입

- 다음 출력 값으로 올바른 것은?

```
var arr = [100, 200, 300];
console.log(typeof(arr));
```

1. undefined
2. string
3. number
4. <code>object</code>

이 문제를 풀려면 **자바스크립트 변수 타입**에 대해서 알아야 한다.

1. 원시타입 : number, string, boolean, undefined, null 👈 이거 외엔 전부 참조타입(object)으로 취급한다.
2. 참조타입 : object

```
var myValue;
console.log(typeof myValue);
//콘솔 결과 : undefined

var myValue = null;
console.log(typeof myValue);
//콘솔 결과 : object
```

변수에 아무것도 없을 때는 undefined가 뜬다. 변수에 null 값을 주면 콘솔 결과에는 object가 뜨는데 사실은 null이 맞다. 근데 왜 object가 찍히냐면 자바스크립트 만든 사람이 처음부터 잘못 만들었다고 한다. 이제와서 고치기엔 너무 많은 사람들이 자바스크립트로 개발을 많이 했기 때문에 그냥 냅둔다고 한다. 그러니 사실상 **var myValue = null 값은 typeof가 null이 맞다.**

### 👉 문제 결과 : 답은 4번이다. (arr 변수는 배열이기 때문에 참조타입인 Object다.)

### 📃 문제 5번 : for문 계산

- 다음 코드의 출력 값으로 알맞은 것은?

```
var a = 10;
var b = 2;

for(var i=1; i<5; i+=2) {
  a+=i;
}

console.log(a+b);
```

[보기]

1. 10
2. 12
3. 14
4. <code>16</code>

### 👉 문제 결과 : 4번 (16)

<br>

위 처럼 보면 분석하기 어려우니 식을 해제를 해보겠다.

🔸 i+=2 이 코드는 i=i+2의 축약이다.

🔸 a+=i; 이 코드는 a=a+i의 축약형이다.

🔸 i 변수는 1이고 i 는 5보다 작다. (i<5) 5와 같아지는 순간 for문은 종료

🔸 i 값은 매번 돌 때마다 2씩 증가한다. (i+=2)

처음에는 i가 1인 상태이고, 두 번째 돌 때는 1(i)+2가 되니
i는 3이 되고 세 번째 돌 때는 i=3+2 되었으니 답은 5가 된다.
그럼 가운데 매개변수 조건문인 i<5는 성립이 되지 않아 for문이 종료된다. 그럼 **최종적으로 for문은 총 2번만 반복된다는 것을 알 수 있다.**

a+=i는 a=a+i이니

🔸 첫 번째 반복 되었을 때 a=10+1 ====> a = 11

🔸 두 번째 반복 되었을 때 a=11+3 ====> a = 14

**결국 a는 14가 되었다.** 그리고 b는 원래 변수에 담겨있던 값 2이니 콘솔에 찍혀 나오는 a+b **최종값은 16이 된다.**

### 📃 문제 6번 : False

- 다음은 자바스크립트 문법 중에서 False로 취급하는 것들 입니다. 앗, False로 취급하지 않는 것이 하나 있네요! Ture를 찾아주세요.

1. NaN
2. <code>1</code>
3. ""
4. 0
5. undefined

### 👉 문제 결과 : 2번 (1)

<br>

자바스크립트에서 False 값은 Falsy / True 값은 Truthy 라고 합니다.

- Falsy : 0, ' ' (빈 문자열), NaN, Null, undefined, False
- Truthy :

```
console.log(!)
```

여기서 느낌표 뒤에 오는 값이 트루면 false를 반환하고, false면 true를 반환한다.
거꾸로 true인지 false인지 반환해준다.
그리고 **숫자가 아니다**라는 것을 반환 받으려면 <code>isNam</code>이라는 메서드를 사용하여 값이 숫자인지 아닌지 확인할 수도 있다.

```
console.log(isNan(1));
```

- Q: 이게 숫자가 아닌 것이 맞니? 이 값이 숫자가 아닌게 맞지?!!!
- A: 아니야 이 안에 있는 것은 숫자가 맞아. 그래서 false값 반환

```
console.log(isNan('abcd'));
```

- isNan 메서드 안에 문자열을 출력하면 True를 반환해준다. (숫자가 아니니까)
